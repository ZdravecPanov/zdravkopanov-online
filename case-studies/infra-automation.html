Infrastructure Automation

Reusable Terraform modules and automation pipelines for consistent environments.

Overview

Designed a modular Infrastructure as Code system to standardize provisioning across environments. The goal was to reduce drift, speed up onboarding, and enforce best practices for security and networking.

Scope: 6 services, 3 environments
Timeline: 6 weeks
Stack: Terraform, Ansible, GitHub Actions
Role: DevOps Engineer

Challenge

Provisioning was manual and inconsistent. Each environment required different steps, and reviewing changes was difficult without a shared IaC baseline.

Approach

  • Built reusable Terraform modules for networking, compute, and security.
  • Implemented environment-specific overlays for staging and production.
  • Automated validation and plan reviews in GitHub Actions.
  • Created runbooks for module usage and rollback workflows.

Architecture

GitHub Actions
   |   
Terraform Plan -> Review -> Apply
   |            
Module Registry
   |
Network + Compute + Observability

Results

  • Reduced provisioning time from hours to minutes.
  • Standardized network policies and security baselines.
  • Improved auditability with Git-based change history.
